
# PRD – Product Requirements Document

## پروژه ساختمان داده – EchoVerse

---

## 1. معرفی پروژه (Project Overview)

**نام پروژه:** EchoVerse
**درس:** ساختمان داده
**هدف کلی:**
طراحی و پیاده‌سازی دستی چند ساختمان داده‌ی اصلی (Tree، Graph، Heap) برای شبیه‌سازی بخش‌های مختلف یک پلتفرم بزرگ پادکست.

این پروژه با هدف **درک عمیق مفاهیم ساختمان داده**، بدون استفاده از کتابخانه‌های آماده، طراحی شده است.

---

## 2. اهداف پروژه (Goals)

* پیاده‌سازی دستی ساختمان داده‌ها بدون استفاده از containerهای آماده
* تسلط عملی بر:

  * درخت جستجوی دودویی خودمتوازن
  * گراف وزن‌دار و الگوریتم دایکسترا
  * درخت و پیمایش BFS / DFS
  * Min-Heap و Heap Sort
* تمرین طراحی کلاس، مدیریت حافظه و مستندسازی

---

## 3. محدودیت‌ها و قوانین (Constraints)

* زبان برنامه‌نویسی: **C++ یا Java**
* استفاده از:

  * `vector`، `array`، `map`، `set`، `priority_queue` ❌ ممنوع
* تمام ساختارها باید:

  * به صورت **دستی**
  * با **کلاس Node مستقل**
  * و با **ارجاع (pointer / reference)** پیاده‌سازی شوند
* پروژه دو نفره است
* مستندسازی و GitHub (Private) الزامی است

---

## 4. فازهای پروژه (Project Phases)

---

### فاز اول: مدیریت کانال‌های تأیید شده

**ساختمان داده:** درخت جستجوی دودویی خودمتوازن (AVL Tree)

#### توصیف

کانال‌هایی که مجاز به انتشار هستند باید در یک درخت جستجوی دودویی خودمتوازن ذخیره شوند.

#### داده‌ها

* `ChannelID` (عدد صحیح یکتا)

#### عملیات مورد نیاز

* `insert(ChannelID)` → تأیید کانال
* `delete(ChannelID)` → لغو تأیید
* `search(ChannelID)` → بررسی مجاز بودن
* `display()` → نمایش ساختار درخت

#### الزامات فنی

* تعریف کلاس `Node`
* تعریف کلاس `AVLTree`
* پیاده‌سازی rotationها به‌صورت دستی
* بدون استفاده از containerها

---

### فاز دوم: پیشنهاد مسیر گوش دادن

**ساختمان داده:** گراف وزن‌دار بدون جهت
**الگوریتم:** Dijkstra

#### توصیف

هر اپیزود یک رأس در گراف است و یال‌ها میزان شباهت یا هزینه ذهنی بین اپیزودها را نشان می‌دهند.

#### وظایف

* ساخت گراف وزن‌دار بر اساس ورودی
* محاسبه کوتاه‌ترین مسیر از یک اپیزود مبدأ
* چاپ:

  * مسیر
  * هزینه نهایی

#### الزامات فنی

* پیاده‌سازی دستی گراف
* پیاده‌سازی دستی الگوریتم Dijkstra
* عدم استفاده از کتابخانه‌های گراف

---

### فاز سوم: ساختار تیم تولید پادکست

**ساختمان داده:** درخت (General Tree)

#### توصیف

ساختار تیم تولید هر پادکست به صورت سلسله‌مراتبی مدل می‌شود.

#### وظایف

* ساخت درخت تیم
* جستجوی عضو با:

  * BFS
  * DFS
* پشتیبانی از:

  * افزودن عضو
  * حذف عضو
  * نمایش ساختار درخت

---

### فاز چهارم: صف انتشار اپیزودها

**ساختمان داده:** Min-Heap

#### توصیف

اپیزودها قبل از انتشار بر اساس اولویت مرتب می‌شوند.

#### داده‌ها

* `EpisodeID`
* `Priority` (عدد کمتر = اولویت بالاتر)

#### عملیات

* `insert(ID, Priority)`
* `extractMin`
* `delete`
* `display`
* پیاده‌سازی Heap Sort
