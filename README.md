
# معرفی پروژه

این پروژه مربوط به درس «ساختمان داده‌ها» بوده و هدف آن پیاده‌سازی دستی مجموعه‌ای از ساختمان داده‌های پایه برای شبیه‌سازی هسته‌ی داده‌ای یک پلتفرم انتشار پادکست به نام **EchoVerse** است.

پروژه شامل چهار فاز مستقل می‌باشد که هر فاز یک مسئله‌ی مشخص را با استفاده از یک ساختمان داده‌ی مناسب حل می‌کند.
در تمام فازها، استفاده از کتابخانه‌های آماده ساختمان داده ممنوع بوده و تمامی پیاده‌سازی‌ها به صورت دستی انجام شده‌اند.

---

## زبان برنامه‌نویسی

Java

---

## ساختار کلی پروژه

پروژه به چهار فاز اصلی تقسیم شده و هر فاز در پوشه‌ی جداگانه‌ای پیاده‌سازی شده است:

* phase1_avl
* phase2_graph
* phase3_team
* phase4_heap

همچنین پوشه‌ی `docs` شامل مستندات هر فاز و فایل‌های توضیحی پروژه می‌باشد.

---

## فاز اول – مدیریت کانال‌های تأیید شده (AVL Tree)

در این فاز، کانال‌های مجاز برای انتشار پادکست در یک درخت جستجوی دودویی خودمتوازن (AVL Tree) ذخیره می‌شوند.

ویژگی‌ها:

* پیاده‌سازی کامل AVL Tree به صورت دستی
* تعریف کلاس مستقل برای Node و Tree
* پشتیبانی از عملیات:

  * insert
  * delete
  * search
  * display
* عدم استفاده از آرایه، لیست، map یا set

هدف این فاز، آشنایی با درخت‌های جستجوی دودویی و عملیات تعادل‌دهی است.

---

## فاز دوم – پیشنهاد مسیر گوش دادن (Graph + Dijkstra)

در این فاز، اپیزودهای پادکست به صورت رأس‌های یک گراف وزن‌دار و بدون جهت مدل شده‌اند.

ویژگی‌ها:

* پیاده‌سازی گراف وزن‌دار بدون جهت
* استفاده از لیست مجاورت به صورت دستی
* پیاده‌سازی الگوریتم دایکسترا بدون استفاده از کتابخانه‌های آماده
* محاسبه کوتاه‌ترین مسیر از یک اپیزود مبدأ
* چاپ مسیر نهایی و هزینه مسیر

---

## فاز سوم – ساختار تیم تولید پادکست (Tree + BFS / DFS)

در این فاز، تیم تولید هر پادکست به صورت یک ساختار سلسله‌مراتبی (درخت) پیاده‌سازی شده است.

ویژگی‌ها:

* هر عضو می‌تواند چند زیردست داشته باشد
* امکان اضافه کردن عضو جدید
* حذف یک عضو به همراه زیر‌درخت آن
* جستجوی اعضا با استفاده از:

  * BFS
  * DFS
* نمایش ساختار درخت تیم تولید

---

## فاز چهارم – صف انتشار اپیزودها (Min-Heap)

در این فاز، اپیزودها قبل از انتشار بر اساس اولویت در یک Min-Heap ذخیره می‌شوند.

ویژگی‌ها:

* پیاده‌سازی Min-Heap به صورت دستی با استفاده از آرایه
* عدد اولویت کوچکتر نشان‌دهنده اولویت بالاتر است
* پشتیبانی از عملیات:

  * insert
  * extract-min
  * delete
  * display
  * heap-sort
* نمایش خروجی‌ها مطابق نمونه‌های داده‌شده در صورت پروژه

---

## نکات مهم

* تمامی ساختمان داده‌ها به صورت دستی پیاده‌سازی شده‌اند
* از هیچ کتابخانه آماده‌ای برای ساختمان داده‌ها استفاده نشده است
* ورودی‌ها داینامیک بوده و به صورت هاردکد نیستند
* کدها ساده، خوانا و در سطح دانشجوی کارشناسی نوشته شده‌اند
* پروژه برای انجام به صورت دو نفره طراحی شده است
* مستندسازی هر فاز به صورت جداگانه انجام شده است

---

## جمع‌بندی

این پروژه شامل پیاده‌سازی عملی و آموزشی ساختمان داده‌های زیر می‌باشد:

* AVL Tree
* Graph و الگوریتم Dijkstra
* Tree Traversal (BFS / DFS)
* Min Heap و Heap Sort

تمامی نیازمندی‌های مشخص‌شده در صورت پروژه به طور کامل پوشش داده شده‌اند و پروژه آماده تحویل نهایی می‌باشد.
